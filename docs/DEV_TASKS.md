### U9 · Column inspector — `partial-rebuild` ✅ (with D4)

Metadata is now a **bordered box of dynamic key/value rows** (`.insp-facts`), plus the title source-format badge and
depth-indented nested rows. The bespoke numeric min/max gradient strip is gone — the design has Min/Max as ordinary
rows. **The real find:** the old Rows/Nulls/Distinct/min/max were computed from `crate::runs` — *the current page of the
current tab's query*. They read as column facts while describing one page of one query, which is what "only real facts"
was pointing at. Deleted. Replaced by the canvas's two-tier model, which interlocks: the box shows **free metadata
only** (footer-derived, `varies by source format; never fabricated`), the PROFILE zone shows what a **scan** computed,
and a fact never appears in both (matched on `StatKey`). Free stats come from DataFusion `Statistics` via
`ListingTable::list_files_for_scan` (footers only, no data pages) → `ColumnInfo.stats: Vec<Stat>` + `CatalogTable.rows`.
⚠️ `ListingOptions::new` hardcodes `collect_stat: false` and a hand-built `ListingTable` never picks up
`datafusion.execution.collect_statistics` — `with_session_config_options` at the register site is what makes that
setting real (and costs a footer read per file). Honesty calls: inexact footer stats (Parquet truncates long strings)
render `~value`; a `null_count == num_rows` is dropped, since DataFusion reports that both for an all-null column *and*
for one with no stats — the profile answers it truthfully instead. Builds here → **D4** ✅.

Completeness bar now needs a real null count (footer or profile) — without one it doesn't render, rather than being
computed off the result page as before.
